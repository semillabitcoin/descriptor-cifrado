# Phase 2 — UI Design Contract

> Visual and interaction contract for the Svelte 5 + Vite 6 SPA embedded via rust-embed.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn not applicable — Svelte 5, not React) |
| Preset | not applicable |
| Component library | none — hand-built Svelte components |
| Icon library | none — inline SVG icons only (zero external requests, <50 KB bundle constraint) |
| Font (body/UI) | Inter variable (woff2 self-hosted in `frontend/src/assets/fonts/`) |
| Font (mono) | JetBrains Mono variable (woff2 self-hosted in `frontend/src/assets/fonts/`) |
| Font fallback | `system-ui, -apple-system, "Segoe UI", Roboto, sans-serif` |
| Mono fallback | `"Cascadia Code", "Fira Code", "Courier New", monospace` |

Source: D-02 (fonts), D-01 (stack context), CLAUDE.md (no CDN constraint).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline badge padding vertical |
| sm | 8px | Button padding vertical, input padding vertical, badge padding horizontal |
| sm+ | 12px | Toast padding vertical, history row padding vertical |
| md | 16px | Default element gap, button/input padding horizontal, card padding, toast padding horizontal |
| lg | 24px | Section gap within a tab panel, between form rows, modal padding |
| xl | 32px | Gap between major sections (form + results zone), drop-zone padding vertical |
| 2xl | 48px | Tab panel top padding, header height internal padding |
| 3xl | 64px | Page max-width side margins on desktop (≥1024px), tab panel bottom padding |

Exceptions:
- Touch targets: minimum 44×44px for all interactive elements (buttons, toggles, file-pick triggers). `min-height: 44px` on buttons/tabs; this overrides the internal padding scale for icon-only controls. Source: D-01.
- Modal content padding: 24px (lg) on all sides.
- Toast width: 320px fixed.
- Border thickness: 1px–2px (not part of spacing scale).
- Focus outline-offset: 2px (not part of spacing scale).

All padding/gap/margin values throughout this document are either declared in the scale above or noted as a documented exception.

## Typography

All sizes in px at 16px root. Line heights are unitless ratios.

**Scale rule:** Inter usa 4 tamaños (14, 16, 20, 28); JetBrains Mono usa 1 tamaño (13px) — excepción de familia monoespaciada para descriptores/código, no compite con la escala Inter.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Inter | 16px | 400 | 1.5 | Paragraphs, labels, descriptions, threat model copy |
| Label | Inter | 14px | 400 | 1.4 | Form labels, tab labels, toggle labels, toast messages, badges |
| Heading | Inter | 20px | 600 | 1.2 | Tab headings, section headings, modal titles |
| Display | Inter | 28px | 600 | 1.1 | App title "BED" in header |
| Mono | JetBrains Mono | 13px | 400 | 1.6 | Descriptor textarea, armored block pre, xpub input, short-id, filename in history |
| Mono bold | JetBrains Mono | 13px | 600 | 1.6 | Highlighted hex/ID fragments in history list |

Notes:
- Inter uses exactly 4 sizes (14, 16, 20, 28) and exactly 2 weights (400 body/label + 600 headings/CTAs/display). Size carries hierarchy; no intermediate weight needed.
- Labels use 14px size for hierarchy differentiation — weight 500 has been removed; 400 is sufficient at this size.
- Display uses 28px + weight 600; the size carries the emphasis — weight 700 has been removed.
- JetBrains Mono is a monospace family exception (13px, 400 + 600) for code/descriptor contexts; its sizes do not count against the Inter 4-size scale.
- Descriptors in textarea use `font-size: 13px` mono to allow long lines to fit visibly on mobile 360px without scrolling horizontally (overflow-x: auto on `<pre>`).
- `<pre>` blocks (armored output, recovered descriptor) use `white-space: pre-wrap; word-break: break-all` to prevent layout overflow.

### 60/30/10 Allocation

- **60% Dominant:** `--color-surface` — page background
- **30% Secondary:** `--color-surface-raised` — all cards, tab panels, modals, result zones
- **10% Accent:** `--color-accent` — reserved for: primary action buttons ("Cifrar", "Descifrar"), active tab underline/indicator, toggle ON state, focus rings, "Modo histórico activo" badge border. NOT used for decorative borders, backgrounds, or text.

### WCAG AA Compliance

All text/background pairs above have been selected to meet ≥4.5:1 for normal text and ≥3:1 for large text and UI components. Source: D-04, D-37.

Theme persistence: `localStorage` key `bed.theme` ∈ `light | dark | auto` (default `auto`). Source: D-03.

## Layout Contract

### Breakpoints

| Name | Width | Behavior |
|------|-------|----------|
| mobile | 360–767px | Single column, full-width inputs, stacked outputs |
| tablet | 768–1023px | Single column with increased side margins (32px) |
| desktop | ≥1024px | Max-width 720px centered, 64px side margins auto |

### Page Structure

```
<body>
  <header>            ← fixed height 56px, surface-raised, border-bottom
    logo + "BED"
    [Modo histórico toggle + badge]
    [Modelo de amenazas button]
    [Theme toggle]
  </header>

  <main>
    <details>         ← Threat model panel (collapsed by default, D-30)
    </details>

    <nav role="tablist">   ← tabs: Cifrar | Descifrar | Historial (conditional)
    </nav>

    <section role="tabpanel">  ← active tab content
    </section>
  </main>
</body>
```

Header items order (left to right): logo "BED" (display 28px) → spacer → history-badge (when ON) → history toggle → threat-model button → theme toggle.

**Focal points:** El focal point primario en estado idle es el textarea del descriptor (Tab Cifrar) o el drop-zone/textarea del armored (Tab Descifrar). En estado success, la zona de resultado (surface-raised, separada por xl=32px) toma el focal point — el executor debe asegurarse de que el scroll lleve esta zona a la vista tras una operación exitosa.

Source: D-05, D-06.

### Tab Panel Widths

All tab panels: `max-width: 640px; margin: 0 auto; padding: 48px 16px 64px`.

On desktop (≥1024px): `padding: 48px 0 64px`.

### Result Zone

The result zone (outputs after encrypt/decrypt) renders below the form in the same tab panel, separated by `32px` (xl). It uses `--color-surface-raised` background, `12px` border-radius, `24px` padding, `1px` solid `--color-border` border.

## Component Contracts

### Buttons

| Variant | Background | Text | Border | Hover | Disabled |
|---------|-----------|------|--------|-------|----------|
| Primary | `--color-accent` | `--color-accent-fg` | none | `--color-accent-hover` | opacity 0.4, cursor not-allowed |
| Secondary | transparent | `--color-text-primary` | 1px `--color-border` | `--color-surface-sunken` bg | opacity 0.4 |
| Destructive | `--color-destructive` | `--color-destructive-fg` | none | `--color-destructive-hover` | opacity 0.4 |
| Ghost | transparent | `--color-text-secondary` | none | `--color-surface-sunken` bg | opacity 0.4 |

Button dimensions: `min-height: 44px; min-width: 44px; border-radius: 8px; padding: 8px 16px` (sm vertical = 8px, md horizontal = 16px). Label font-size 14px, weight 400 (Label role). The `min-height: 44px` touch-target exception ensures the button meets the touch target requirement even though the padding alone gives less than 44px vertical.

Loading state: spinner replaces leading icon, label changes to active verb ("Cifrando…" / "Descifrando…" / "Guardando…"), button disabled. Source: D-33.

Copy button post-action: label changes to "Copiado ✓" for 1500ms, then reverts. Toast also fires. Source: D-34.

### Inputs and Textareas

```
border: 1px solid --color-border
border-radius: 8px
background: --color-surface-sunken
padding: 8px 16px
font-size: 16px (body inputs) / 13px mono (descriptor, xpub, armored)
focus: border-color --color-border-focus, box-shadow 0 0 0 3px rgba(accent, 0.2)
```

Textarea for descriptor / armored: `min-height: 120px; resize: vertical; font-family: JetBrains Mono`.

Textarea for xpub: `min-height: 72px; resize: vertical; font-family: JetBrains Mono`.

Every input has an associated `<label for>` and error message wired via `aria-describedby`. Source: D-37.

### Tabs

```
role="tablist" — horizontal strip, full panel width
role="tab" — aria-selected, aria-controls
  active: border-bottom 2px solid --color-accent, font-weight 600
  inactive: border-bottom 2px solid transparent, font-weight 400
  focus: outline 2px solid --color-border-focus, outline-offset 2px
  height: 44px (touch target)
role="tabpanel" — id matching tab aria-controls
```

Historial tab: rendered in DOM only when `bed.historyEnabled === true`. Source: D-20.

### Toggle (History Mode)

Visual: pill toggle, 44px wide × 24px tall. ON state: `--color-accent` background. OFF state: `--color-border` background. Thumb: white circle 18px, transition `left 150ms ease`. Label "Historial" to the left, 14px weight 400.

ARIA: `role="switch" aria-checked="true|false"`. Source: D-18.

Badge "Modo histórico activo": appears in header when toggle ON, `--color-history-badge-bg` background, `--color-history-badge-text` text, `border-radius: 999px`, `padding: 4px 8px` (xs vertical = 4px, sm horizontal = 8px), `font-size: 14px weight 400` (Label role). Source: D-18.

### Threat Model Panel (`<details>`)

Trigger: `<summary>` styled as ghost button with ⚠ icon (inline SVG) + "Modelo de amenazas" text. Positioned in header. Source: D-30, D-32.

When expanded: panel renders below header as a full-width banner, `z-index` layered above tab content. Padding: 24px. Contains:

1. Warning callout box: `--color-warning-bg` background, `--color-warning-border` left border 4px, `--color-warning-text` text. Copy: *"Ninguna ubicación debe contener simultáneamente el `.bed` y una xpub del multisig."* (bold, 16px body).
2. "Lo que protege" block: `--color-success-bg`, `--color-success-text`, prefix "✓ Protege:".
3. "Lo que NO protege" block: `--color-caution-bg`, `--color-caution-text`, prefix "✗ No protege:".
4. Link al README (texto "Ver modelo completo →", ghost button style).

Source: D-31.

### Drop-zone (Descifrar tab)

```
border: 2px dashed --color-border
border-radius: 12px
background: --color-surface-sunken
padding: 32px 16px
text-align: center
```

Drag-over state: `border-color: --color-accent; background: rgba(accent, 0.04)`.

Accessible: `role="button" tabindex="0"` with keyboard activation (Enter/Space opens file picker). Source: D-13.

### Modal (Confirmación borrar)

Backdrop: `rgba(0,0,0,0.4)` with blur `backdrop-filter: blur(2px)`.
Panel: `--color-surface-raised`, `border-radius: 12px`, `padding: 24px`, `max-width: 400px`, centered via flex.
Focus trap: first focus on "Cancelar" button (default focus). Source: D-36.
Escape closes modal (cancels action).

### Toast

Position: top-right, `z-index: 9999`, `margin: 16px`.
Dimensions: `width: 320px; padding: 12px 16px; border-radius: 8px` (sm+ vertical = 12px, md horizontal = 16px).
Background: `--color-toast-bg`. Text: `--color-toast-text` 14px weight 400.
Auto-dismiss: 3000ms. Animation: slide-in from right 200ms ease-out, slide-out 150ms ease-in.
`aria-live="polite"` on toast container. Source: D-34.

### Inline Error Alert

Position: immediately above the relevant form, full width of panel.
Style: `--color-warning-bg` background, `--color-warning-border` left border 4px, ⚠ icon + message text, × close button (ghost).
`role="alert"` for immediate screen reader announcement. Source: D-35.

### `<pre>` Result Blocks

```css
font-family: JetBrains Mono, monospace;
font-size: 13px;
line-height: 1.6;
background: --color-surface-sunken;
border: 1px solid --color-border;
border-radius: 8px;
padding: 16px;
white-space: pre-wrap;
word-break: break-all;
overflow-x: auto;
max-height: 200px;
overflow-y: auto;
```

### History Entry Row

Layout: horizontal flex, `min-height: 56px`, `padding: 12px 16px`, `border-bottom: 1px solid --color-border`.

```
[timestamp relative]   [filename short-id mono]   [Ver]  [Borrar]
```

Timestamp: 14px secondary text + tooltip (title attribute) with ISO absolute. Source: D-21.

### Spinner

Inline CSS `border` spinner, 16px × 16px, border 2px, `--color-accent-fg` on primary button, `--color-accent` on ghost/secondary. Animation: rotate 600ms linear infinite.

## Accessibility Requirements (WCAG AA)

| Requirement | Implementation |
|------------|---------------|
| Label association | `<label for="id">` on every input; `aria-label` on icon-only buttons |
| Semantic HTML | `<header>`, `<main>`, `<nav>`, `<section>`, `<button>` — no `<div onclick>` |
| Contrast | All token pairs meet ≥4.5:1 normal text, ≥3:1 large/UI |
| Focus visible | `outline: 2px solid --color-border-focus; outline-offset: 2px` on all interactive elements (never removed, only styled) |
| Tab order | Logical DOM order: header → threat model → tabs → tab panel → results |
| Error association | Error messages linked via `aria-describedby` to their input |
| Live regions | `aria-live="polite"` on toast container; `role="alert"` on inline error alerts |
| Icon-only controls | `aria-label` on theme toggle, close ×, and icon-only buttons |
| Color independence | Every state uses icon + text, never color alone |
| Tab ARIA | `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls`, `role="tabpanel"`, `aria-labelledby` |
| Modal | Focus trap while open; Escape closes; `role="dialog"` + `aria-modal="true"` + `aria-labelledby` |
| Details/summary | Native `<details>`/`<summary>` or equivalent with `aria-expanded` |
| History toggle | `role="switch"` + `aria-checked` |
