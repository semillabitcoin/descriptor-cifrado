# Quick Task 260508-m7g: Fix refresh historial v0.1.5

## Problema

Tras `POST /api/encrypt` + `POST /api/history` exitosos, la pestaña Historial NO muestra la entrada nueva hasta `Ctrl+Shift+R`. Causa: en `App.svelte:38-49` los tres `<section>` permanecen montados (solo togglea `hidden`); por tanto el `$effect(() => { void loadList(); })` de `TabHistorial.svelte:25` corre una sola vez al montar la app y nunca se vuelve a disparar al cambiar de pestaña ni al cifrar.

## Estrategia

Añadir un signal reactivo `historyVersion` en el store global. TabCifrar lo incrementa tras éxito de `POST /api/history`. TabHistorial lee el signal dentro de su `$effect` para que se vuelva reactivo y recargue la lista cada vez que cambia.

Mismo patrón se aplica también cuando TabHistorial borra una entrada (mantiene consistencia entre múltiples pestañas activas / future hooks). Pero eso ya funciona porque la fila se elimina localmente. Mantener cambios mínimos: solo el signal + incremento desde TabCifrar + lectura desde TabHistorial.

## Archivos a tocar

1. **`frontend/src/stores/app.svelte.js`** — añadir `historyVersion: 0` al `appState` y exportar helper `bumpHistoryVersion()`.
2. **`frontend/src/components/TabCifrar.svelte`** — tras `postJson('/api/history', ...)` exitoso (línea ~91), invocar `bumpHistoryVersion()`.
3. **`frontend/src/components/TabHistorial.svelte`** — el `$effect` debe leer `appState.historyVersion` para volverse reactivo a cambios.

## Validación

- `npm run build` en `frontend/` sin errores ni warnings nuevos.
- Smoke: con server local + history ON, cifrar dos descriptors seguidos sin refrescar y verificar que ambos aparecen en la pestaña Historial al cambiar a ella.
